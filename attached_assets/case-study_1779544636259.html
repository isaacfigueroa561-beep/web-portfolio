The challenge

Fintech that treats you like an adult.

The tools available to people with $100k+ in investable assets are stuck between two bad extremes — neither actually fits.

01
Too clinical, or too gamified
Schwab feels like tax software. Robinhood treats investing like a slot machine. Neither respects the user's intelligence or taste.
02
Advisors are for $2M+ clients
Human financial advisors are out of reach. AI alternatives either feel like chatbots or are buried under 12-step onboarding flows.
03
Beautiful ≠ useful
Apps that look good tend to optimize for engagement over outcomes — dark patterns and notification addiction don't grow wealth.
"The goal was to build the app that feels like it was made by someone who already figured money out — not someone trying to sell you a product."

Key screens

Three moments, one product.

meridian.app/home
1
Live net worth counter

Numbers count up from $0 on load. A live dot + 3-second micro-tick simulate real market data to reinforce the "always on" feel.

2
Animated SVG chart

The path draws left-to-right via stroke-dashoffset. Six time periods each re-trigger the animation from scratch for a fresh reveal every time.

3
Marisol — AI advisor

Contextual nudges in the right moment — after detecting idle cash, a concentration risk, or an actionable opportunity. Advisor, not chatbot.

4
Staggered card entrance

Every screen entry triggers a choreographed fade-up cascade — 60ms offsets per card so nothing feels mechanical or staged.

meridian.app/portfolio
1
Holdings table

Hover states expand the row with background fill — a subtle spatial cue that this row is interactive without adding any chrome.

2
Concentration warning

Marisol detects NVDA at 11% of the portfolio and surfaces a rebalance prompt in context — not as a push notification, but a card.

3
Portfolio stats

Cost basis, unrealized gain, dividend yield, Sharpe ratio — the numbers a real investor wants, without a separate analytics screen.

meridian.app/learn
1
Editorial masthead

The Meridian Weekly feels like a financial magazine — issue number, serif display headline, italic subheading. Inspired by The Economist and Bloomberg Pursuits.

2
Contextual reading

Lead essay chosen based on the user's actual situation — idle bonus, looming tax season, new income bracket. Content as a feature, not a feed.

3
Read-time accent bars

Each article card has a thin green bar that fills proportionally to reading time — a small visual signal that respects the user's attention budget.

Feature highlights

The details that make it feel real.

Animated SVG charts
Every chart draws itself in using stroke-dashoffset animation — 1.8s of smooth cubic easing on a Catmull-Rom spline path. Fill fades in 400ms later so the stroke lands first.
CSS Animation
Number counters + live ticks
Net worth counts up from $0 using requestAnimationFrame with ease-out-cubic. A setInterval hook updates the value every 3.2s with a small random walk to simulate live market data.
React Hooks
Dark mode
A full warm-dark palette — not just inverted. Deep walnut tones replace the parchment. A toggle with a sliding knob + CSS custom properties ensures every component adapts instantly.
CSS Variables
Chart hover tooltip
A crosshair + value bubble tracks the cursor across the chart. The SVG mousemove handler maps pixel coordinates to data indices in real-time — no external chart library required.
Vanilla SVG
Marisol AI advisor
Contextual, proactive nudges that appear in the right moment. Framing it as "advisor" vs "chatbot" is a deliberate UX decision — she acts on your behalf, not on command.
Interaction Design
Splash screen
A CSS-only animated loading screen with a spinning brand mark and progress bar masks the Babel compile time. The app appears only when fully ready — no flash of unstyled content.
CSS + React
How it was built

From wireframe to live app.

Discovery
Research & positioning

Audited 12 fintech products from Robinhood to Wealthfront to JP Morgan Private. Identified the white space: $150k–$400k earners in accumulation phase who value taste over features. Defined the "quiet wealth" positioning and the Marisol persona.

Design
Tokens & design system first

Started with the three-font system (Instrument Serif + Onest + JetBrains Mono) — each earns its role and nothing overlaps. Then palette, spacing scale, and component primitives. The system came before any screen layout.

Prototype
Hi-fi prototyping in Figma

All three screens built to full fidelity in Figma — layout, typography, colour, and component states — before writing a line of application code. Explored multiple directions fast, then locked the one direction that felt right and moved on.

Build
Design to production

I built the live app myself, translating the Figma mockups into a single dependency-minimal React file. Every layout decision, animation curve, and dark-mode variant was hand-coded — no libraries, no shortcuts, just component logic and CSS custom properties.

Polish
Animation layer

Every animation was added after the layout was solid: chart draw, counters, donut sweep, progress bars, staggered entrance. Layout-first, motion-second — so animation enhances instead of masking structural issues.

Tech stack

Intentionally minimal.

No build step. No bundler. No dependencies beyond React. The goal was to prove the design and interactions — not engineer for scale.
